# Monsoon Ready

A monsoon preparedness & citizen assistance micro-app. Enter a city, household size, and
language — get a personalized preparedness plan grounded in **real, live weather data**
and generated by a **real LLM call**, translated into your selected language.

**Live URL:** https://monsoon-ready.vercel.app

## What it does

1. Takes location (city), household size, and language preference as input.
2. Fetches real current conditions + a 7-day forecast for that city from
   [Open-Meteo](https://open-meteo.com/) (free, no API key).
3. Computes a severity level (`low` / `moderate` / `severe` / `extreme`) from the forecast
   using IMD-style rainfall and wind thresholds, and shows an alert banner when thresholds
   are crossed (simulated real-time alerting — no push infrastructure).
4. Calls Gemini (`gemini-flash-latest`) with the real forecast and a deterministically
   computed base checklist (so quantities are never hallucinated) to generate:
   - A before / during / after preparedness plan
   - A household-size-tailored emergency checklist
   - Safety recommendations calibrated to the forecast severity
   - A one-line travel advisory
   - All of the above translated into the selected language
5. Displays everything on one results view.

## Tech stack

- Next.js 16 (App Router) + TypeScript + Tailwind CSS
- Weather: Open-Meteo geocoding + forecast REST APIs
- LLM: Google Gemini (`gemini-flash-latest`) via the Generative Language REST API
- Testing: Vitest (unit tests for severity thresholds + checklist logic), Playwright (E2E)

## Setup

```bash
npm install
cp .env.local.example .env.local
# add your GEMINI_API_KEY (free: https://aistudio.google.com/apikey)
npm run dev
```

Open http://localhost:3000.

## Testing

```bash
npm test        # unit tests (lib/weather.ts, lib/checklist.ts)
npm run test:e2e  # Playwright E2E — hits the real weather + LLM APIs end-to-end
```

## Notes

- `GEMINI_API_KEY` is read server-side only (in `app/api/plan/route.ts` and `lib/llm.ts`)
  and is never sent to the client.
- No login/auth. This scaffold includes an unused Supabase client
  (`lib/supabase.ts`, `NEXT_PUBLIC_SUPABASE_*` env vars) left over from the starter
  template — this app doesn't persist any data, so it isn't wired up.
- Deployed to Vercel: after adding `GEMINI_API_KEY` locally, it must also be added via
  the Vercel dashboard (or `vercel env add`) and the project redeployed for it to take
  effect in production.
